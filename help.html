AutoFiller Documentation

Everything you need to know to harness the power of AutoFiller.

1. Getting Started

AutoFiller is designed to expedite local development and form testing by saving you from monotonous typing.

  • Step 1: Add an Application group to keep your configurations organized.
  • Step 2: Add a Target URL Rule. This dictates on which websites the autofill will be active.
  • Step 3: Add Field Mappings by providing the exact CSS Selector of the inputs on the page, and the content you'd like inserted.

2. URL Match Types

When creating a rule, you must select how the extension identifies the current web page.

Exact Match: Matches the exact string character by character. If the URL has query parameters like ?id=123, it will fail unless mapped explicitly.

Starts With: Excellent for matching a domain structure (e.g. http://localhost:3000/).

Wildcard (*): The most powerful option. Replaces any variable data with a star. Use this for dynamic ports and dynamic routing paths. Example: http://localhost:*/api/v1/*/edit.

3. CSS Selectors (How to target fields)

A CSS Selector is the exact address of an HTML element on the page. You can easily find the CSS selector by right-clicking any input box in Chrome, selecting "Inspect Element", and looking at its attributes.

Type Format Example HTML
By ID (Best) #element-id <input id="username"> → #username
By Class Name .class-name <input class="email-field"> → .email-field
By Name Attribute input[name="value"] <input name="first_name"> → input[name="first_name"]
By Type Attribute input[type="email"] <input type="email"> → input[type="email"]
Nested Selector .parent .child <div class="form"><input id="pass"></div> → .form #pass
